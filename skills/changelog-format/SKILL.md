# Changelog Format

Produce a concise Markdown changelog:

1. An `#` H1 line: `## Changelog` or the release tag.
2. A bulleted list — one `-` item per approved feature, newest first.
3. Each item: a short imperative phrase (e.g. "Add admin-configurable onboarding
   checklists") followed by a one-sentence user-facing benefit.

No marketing prose, no metrics, no internal codenames. One line per feature.
