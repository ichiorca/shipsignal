# Battlecard Delta Format

Produce a sales battlecard DELTA in Markdown — only what changed this release, not a full
battlecard:

1. An `#` H1: "Battlecard delta — <release or feature theme>".
2. `## What changed` — one `-` bullet per approved feature: the capability in seller
   language (what you can now say on a call).
3. `## Talking points` — 2-3 bullets connecting the change to buyer pain. Grounded in the
   approved features only; no invented competitor claims, win rates, or customer quotes.
4. `## Watch-outs` — 1-2 bullets on limits or rollout caveats sellers must not overpromise
   (only if the approved features state them; otherwise omit the section).

Under 200 words. Never name competitors unless the approved features name them. No
pricing, discounts, metrics, or availability dates unless they appear in the approved
features. No internal codenames.
