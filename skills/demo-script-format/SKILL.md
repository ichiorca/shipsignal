# Demo Script Format

Produce a demo video script and screen-flow plan as Markdown:

1. An `#` H1 naming the demo theme.
2. A one-paragraph `## Setup` — the scenario and the user persona, drawn from the
   approved feature audiences.
3. A `## Screen flow` numbered list. Each step is one on-screen action paired with
   the narration line a presenter would say. Steps must describe only flows that the
   approved features actually deliver — no imagined screens or unreleased UI.
4. A short `## Wrap-up` line summarizing the user value shown.

Keep narration lines short and speakable. Use only the supplied feature details. Do
not fabricate metrics, customer names, screens, or availability dates. Every claim a
narration line makes must trace back to an approved feature so it survives claim checks.
