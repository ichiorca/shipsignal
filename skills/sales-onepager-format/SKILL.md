# Sales One-Pager Format

Structure a sales one-pager as Markdown for a single approved release:

1. An `#` H1 with the value proposition in one line — the outcome a buyer gets.
2. A short `## Who it's for` section naming the audiences from the supplied features.
3. A `## Use cases` bulleted list — one concrete, evidence-backed use case per feature.
4. A `## Objection handling` list — anticipated objections with a factual response.
   Never invent competitor claims, pricing, or guarantees.
5. A `## Talk track` — three or four short talking points a rep can say verbatim.

Use only the supplied feature details. Do not fabricate metrics, ROI figures,
customer names, or availability dates. Keep it under ~400 words.
