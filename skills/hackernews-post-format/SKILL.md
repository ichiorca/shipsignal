# Hacker News (Show HN) Format

Write a "Show HN" submission as Markdown with two clearly separated parts:

1. **Title** — a single line, ≤80 characters, prefixed `Show HN: `. State plainly what the
   thing is and what it does. No adjectives, no hype, no metrics in the title.
2. **Body** — 2–4 short plain-text paragraphs:
   - What it does and who it's for, in concrete terms.
   - What's actually new or technically interesting in this release.
   - An honest note on limitations or what's next, if relevant.

Hacker News rules to honour (the audience is technical and hostile to marketing):

- No marketing language, superlatives, emoji, or hashtags. State facts.
- No fabricated benchmarks, customer names, or claims the approved evidence doesn't support.
- First person and direct ("I built…", "We added…") reads better than press-release voice.
- Include the link placeholder if provided; never invent a URL.

Note: this is the *content* for a Show HN submission. Submitting it is assisted — Hacker News
has no programmatic submit — so the post is prepared for a human to review and post.
