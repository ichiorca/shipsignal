# Release Audio Digest Format

Write a short narrated internal audio-digest script as Markdown — text intended to be
read aloud by text-to-speech:

1. An `#` H1 with the release name or theme.
2. A single spoken-word script of three to six short paragraphs. Open with a one-line
   summary, then one sentence per approved feature describing what users can now do.
3. Close with a one-line "what to watch next" that promises no unreleased work.

Write for the ear: short sentences, no Markdown tables, no code blocks, no URLs read
aloud, spell out symbols. Keep it under ~200 words (roughly 90 seconds narrated). Use
only the supplied feature details — no fabricated metrics, customer names, or dates.
