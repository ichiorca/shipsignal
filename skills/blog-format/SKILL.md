# Release Blog Format

Structure a release blog post as Markdown:

1. An `#` H1 headline naming the release theme.
2. A one-paragraph intro: who benefits and why it matters.
3. One `##` H2 section per approved feature — what it does, the user value, who it
   is for. Use only the supplied feature details.
4. A short closing paragraph. No call-to-action that promises unreleased work.

Keep it under ~500 words. Do not fabricate quotes, numbers, or availability dates.
